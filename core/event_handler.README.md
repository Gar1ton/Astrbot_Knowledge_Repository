# core/event_handler — 事件分发层

> 真实项目中体现为 `core/event_handler.py`。本文件说明其职责与边界。

## 职责

接收来自薄壳层（`main`）转发的**框架事件**，决定该调用哪个子系统，然后**委派**。
它是「框架世界」与「业务世界」之间的**路由决策点**——让 `main` 保持零业务，让 managers 保持框架无关。

## 它做什么 / 不做什么

| 做 | 不做 |
|----|------|
| 判断事件类型、取出/清洗框架字段（经 adapters） | 不实现具体业务算法（交给 managers/pipelines） |
| 决定路由到哪个 manager / 方法 | 不直接读写存储（交给 repository，经 manager） |
| 把框架数据翻译成 `domain` 入参 | 不在此堆叠条件分支式的业务规则 |

## 约定

- 接收**组合根注入的依赖**（通常持有 initializer 或各 manager 引用），自身不构造依赖。
- 取框架字段用「安全取值」helper，隔离框架 API 的多变性。
- 路由分发逻辑可以有，但**业务计算一律下沉**到 managers/pipelines。
- 单文件偏大时，按事件类别拆分多个 handler（保持单一职责）。
