# 知识库控制台 · 视效优化落地说明（Handoff v2）

> 面向**本地开发 Agent / 工程师**。本文件描述在已确认的「暖色 Fumadocs」基线之上，本轮**视效增强 + 交互细化**的全部改动，并给出**落地到生产前端（`web/frontend/` Next.js）的映射**。
>
> ⚠️ **硬约束（务必遵守）**
> 1. **后端端口契约完全不变**：所有网络访问仍走 `lib/api.ts` 的统一出口（对应原型 `data.jsx` 的 `window.KR.*`），路径 / 方法 / 字段 / 鉴权与 `design-spec.md §6` 保持一致。本轮**没有新增或修改任何 API**。
> 2. 视效层为**纯前端展示层**（`pointer-events: none`），不得阻塞交互、不得影响数据流。
> 3. 配色一律由设计令牌驱动，**禁止**在组件内写死颜色。

参考实现：交互式高保真原型位于 `console/`（React + Babel 单文件直跑，零构建）。下述每条改动都可在原型对应文件中取样式与逻辑。

## 1. 设计令牌（`styles/tokens.css`）

### 1.1 强调色：可调 HSL
强调色由三个变量驱动，**派生色用 `color-mix` 级联**，因此换肤只需改 H/S/L：

```css
:root{
  --accent-h: 32;          /* 色相 0–360 */
  --accent-s: 80%;         /* 饱和度 */
  --accent-l: 48%;         /* 明度 */
  --accent:        hsl(var(--accent-h) var(--accent-s) var(--accent-l));
  --accent-strong: color-mix(in srgb, var(--accent) 80%, #1a1206); /* 跟随 H/S/L，不再写死亮度 */
  --accent-fg: #fff;

  --accent-soft:   color-mix(in srgb, var(--accent) 14%, var(--surface));
  --accent-softer: color-mix(in srgb, var(--accent)  7%, var(--surface));
  --accent-border: color-mix(in srgb, var(--accent) 42%, var(--border));
  --ring:          color-mix(in srgb, var(--accent) 32%, transparent);
  --glow:          color-mix(in srgb, var(--accent) 30%, transparent);
}
.dark{
  --accent-h: 30; --accent-s: 82%; --accent-l: 54%;
  --accent-strong: color-mix(in srgb, var(--accent) 78%, #fff);
  /* soft / border / ring / glow 同浅色思路，混入比例略高，见原型 */
}
```

**落地要点**：把现有 `next-themes` 主题色接到这三个变量；设置页写入时同时 `document.documentElement.style.setProperty('--accent-h' | '-s' | '-l', …)` 并持久化（原型用 `localStorage` 键 `kr-hue` / `kr-sat` / `kr-light`，生产可改用偏好接口或保留本地存储）。

### 1.2 视效工具令牌
```css
:root{
  --glow: …;                              /* 强调色光晕（跟随光圈、按钮 hover 用） */
  --shadow-sm / --shadow / --shadow-lift / --shadow-pop;  /* 暖调分层阴影 */
}
```
> 上一版引入过 `--neu-*`（新拟物）令牌，**本轮 C4 改为扁平毛玻璃后已不再用于图谱节点**；如其它处未使用可移除。

## 2. 全局视效层（`components/fx/*` → 原型 `fx.jsx`）

统一封装为 `<Atmosphere variant sun follow>`，置于每个页面最底层（`position:absolute; inset:0; z-index:0; pointer-events:none`）。包含：

### 2.1 跟随鼠标光晕（C1，本轮重点）
- 一个柔光圆斑，**在所属页面范围内随鼠标中心缓动移动**（线性插值 `lerp ≈ 0.06`，带惯性）。
- **参数（已按用户反馈调淡调小）**：`width/height = 360px`、整体 `opacity = 0.5`、`filter: blur(8px)`、径向渐变：
  ```
  radial-gradient(circle at 50% 50%,
    var(--glow) 0%,
    color-mix(in srgb, var(--accent) 5%, transparent) 40%,
    transparent 68%)
  ```
- 实现：`requestAnimationFrame` 中读取 `mousemove` 归一化坐标（相对 host 矩形），写 `glow.style.left/top = cx*100% / cy*100%`。
- **降级**：`prefers-reduced-motion: reduce` 时停止跟随与视差。

### 2.2 其它氛围层
- **极光 Aurora**：2–3 个大柔光斑缓慢漂移（`auroraShift`，CSS keyframes）。
- **漂浮点场 DotField**：约 22 个强调色小点，错峰漂移 + 明灭，近景点带光晕。
- **分层视差**：`.fx-layer[data-depth]` 随鼠标用 `style.translate` 偏移（与各层自身动画用 `translate` 属性合成，互不覆盖）。
- **纸张颗粒 GrainOverlay**：`feTurbulence` 噪声，`mix-blend-mode: overlay`，`opacity ≈ .05`，`position:fixed` 顶层。
- **半调太阳 SunBloom**：canvas 点阵 + 缓旋呼吸——**仅登录页保留**作为 Hero；应用内各页已移除（避免与跟随光圈重复）。

## 3. 各界面要点

### 3.1 Ask Agent（`app/(console)/ask/page.tsx` → `pages-ask.jsx`）
- **C2：删除空状态的示例问题（`ask_s1/2/3` 建议气泡）**，且不再提供该功能。空状态 = 图标 + 标题 `ask_empty_title` + 说明 `ask_empty_sub`，居中。
- 保留：对话气泡、`embedding + RRF` 标识、`新对话`、集合范围选择、圆形发送按钮（按压回弹）。
- **引用联动**：回答正文内 `[n]` 角标与右侧来源卡 hover 双向高亮（角标 ↔ 卡片同时点亮）。来源卡含标题、片段、`RRF` 分值、「在文档中打开」跳转。
- 端口：`POST /api/ask`（字段 `question / collection / top_k / conversation_id`，见 §6）。

### 3.2 文档工作台（`documents/page.tsx`）
四区布局：集合/标签筛选 · 文档表格（彩色扩展名徽标 / 标签 / 大小 / 时间，可排序）· 右侧检查器（元信息、改集合、编辑标签、分块预览、下载/删除）；多选批量移动/打标签/删除。端口：`/api/collections`、`/api/documents`（GET/POST/PATCH/DELETE）。

### 3.3 知识库检索（`search/page.tsx`）
集合 + 关键词 + Top-K；结果卡片含命中高亮、`chunk` 序号、`RRF` 标识。端口：`GET /api/kb/search`。

### 3.4 知识图谱（`graph/page.tsx` → `pages-graph.jsx`）★ 本轮重做
**C4 节点最终形态 = 扁平淡毛玻璃圆盘**（HTML 节点 + SVG 连线 hybrid）：

- 容器：相对定位、`aspect-ratio: 560/440`；SVG 连线层 `position:absolute; inset:0; width/height:100%; viewBox 0 0 560 440; preserveAspectRatio="none"`，HTML 节点用百分比定位（`left = x/560*100%`、`top = y/440*100%`）对齐。
- **节点圆盘样式（关键）**：
  ```
  border-radius: 50%;
  background: color-mix(in srgb, {typeColor} {选中?34:20}%, color-mix(in srgb, var(--surface) 52%, transparent));
  backdrop-filter: blur(7px);
  border: 1px solid color-mix(in srgb, {typeColor} {选中?72:40}%, transparent);
  box-shadow: {选中? 0 0 0 3px color-mix(in srgb,{typeColor} 20%, transparent) : none};
  /* 无高光层、无双向阴影、无立体浮雕 */
  ```
  节点直径 `34 + degree*8` px；标签置于圆盘下方。
- **连线**：SVG `line`，`vector-effect: non-scaling-stroke`（防 `preserveAspectRatio=none` 拉伸描边）；另叠一条透明粗线作点击热区。
- **关系标签**：扁平毛玻璃小药丸（`background: color-mix(in srgb, var(--surface) 70%, transparent); backdrop-filter: blur(6px); border:1px solid var(--border)`），**仅在该边处于聚焦集时显示**，避免杂乱。
- **交互**：hover/选中某节点 → 高亮其邻域、其余 `opacity:.34`；选中节点描边强调色 + 轻微放大；点击画布空白处取消选择。
- **类型配色** `typeColor`：`Method/Algorithm → var(--accent)`、`Dataset → var(--accent-2)`、`Concept → 蓝`、`Person → 紫`。
- 端口：`GET /api/graph`、`GET /api/graph/query`、`POST /api/graph/build`（保留 reserved 透传）。下方「图谱增强查询」展示召回诊断（vector / keyword / graph 命中 + RRF 条形）。

### 3.5 配额（`quota/page.tsx`）
R2 / Notion 用量卡，进度条含 80% 阈值刻度，按 `ratio` 给出 正常/警告/危险 文案与配色。端口：`GET /api/quota`。

### 3.6 同步 / 备份（`sync/page.tsx`）
Notion 镜像 / R2 同步 / 备份恢复分区；`reserved` 操作显示「即将上线」并透传 `available_in`。端口：`/api/notion/init`、`/api/sync/notion/pull`、`/api/sync/r2`、`/api/backup`、`/api/restore`。

### 3.7 设置（`settings/page.tsx` → `pages-ops.jsx`）★ 本轮增强
- **外观**：浅/深主题、语言（中/英）分段控件。
- **C3 强调色**：色相轮 + **三条渐变滑杆「色相 / 饱和度 / 明度」**（轨道实时反映当前色彩）+ 6 个预设 + `hsl(h s% l%)` 实时读数。
  - 滑杆轨道渐变：
    - 色相：`linear-gradient(90deg, hsl(0 s l) … hsl(360 s l))`
    - 饱和度：`linear-gradient(90deg, hsl(h 0% l), hsl(h 100% l))`
    - 明度：`linear-gradient(90deg, hsl(h s 24%), hsl(h s 50%), hsl(h s 78%))`（明度建议范围 24–78，避免极端）
  - 三值写入 `--accent-h/-s/-l` 并持久化，全站实时跟随。
- **后端有效配置（只读核对）**：`GET /api/config/effective`，敏感字段后端已打码（`****`）原样显示。

## 4. 交互与动效细则
- **进场**：内容基线可见，支持动效时 `fadeUp` 错峰（`prefers-reduced-motion` 降级为 `fadeIn`）。
- **悬停抬升** `.lift`、**按压回弹**（按钮 `scale(.96)`）、列表行/卡片 hover 反馈。
- **引用 ↔ 来源联动**、**图谱邻域聚焦**、**配额阈值刻度**。
- 性能：跟随光圈与视差合并在单个 rAF 循环；视效层 `pointer-events:none`、`will-change` 谨慎使用。

---

## 5. 主题 / 语言 / 换肤
- 深色：`<html class="dark">`（对齐 `next-themes`）。
- 语言：`zh` / `en` 字典（原型 `i18n.jsx`，对应生产 `lib/i18n.ts`），新增键：`settings_hue` / `settings_saturation` / `settings_lightness` 及更新后的 `settings_accent_hint`。
- 换肤：改 `--accent-h/-s/-l` 即全站级联，无需逐组件调整。

---

## 6. 端口契约（不变，照抄 design-spec §6）
所有调用经统一出口（生产 `lib/api.ts`）。本轮**未改动**任何端口。要点回顾：
- 文档/集合：`/api/documents`、`/api/collections`
- 检索：`/api/kb/search`、`/api/kb/collections`
- 图谱：`/api/graph`、`/api/graph/query`、`/api/graph/build`
- 配额/配置：`/api/quota`、`/api/config/effective`
- 同步/备份（部分 reserved 透传）：`/api/notion/init`、`/api/sync/notion/pull`、`/api/sync/r2`、`/api/backup`、`/api/restore`
- 鉴权：`/api/auth`、`/api/login`
- Ask：`POST /api/ask` → `{ question, collection, top_k, conversation_id }`，返回 `{ conversation_id, answer, sources[] }`

> 原型默认走内置 mock（离线可演示）；URL 加 `?live` 即切真实同源后端，端口照旧。
